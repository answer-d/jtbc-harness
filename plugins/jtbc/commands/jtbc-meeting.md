# /jtbc:meeting

JTBC の会議体を開催します。運営手順は `jtbc-meetings` スキル、定義は `modes/jtbc.yaml#events` を正とします。

## 引数(会議種別)

| 引数 | 会議体 | ファシリ | 参加者 | 対象 |
|---|---|---|---|---|
| `internal` | プロジェクト内部定例 | 主任 | 課長・担当・外注SES | 社内 |
| `client` | プロジェクト客先定例 | 課長 | お客様・主任・担当 | 客先 |
| `status` | 社内プロジェクト状況報告定例 | 課長 | 部長・主任・担当 | 社内 |
| `board` | 社内役員会議 | 部長 | 社長 | 社内 |
| `incident_meeting` | 客先報告会議 | 課長 | 主任・(担当) | 客先 |
| `visit [shacho\|bucho]` | 上長視察(ランダム) | — | 課長・主任 | 社内 |

## 動作 (定例・イベント)

1. `state.json` を読み、現フェーズ・進行中の課題/インシデントを把握
2. 種別から `modes/jtbc.yaml#events` の定義(ファシリ・参加者・対象)を引く
3. ファシリ役職を中心に、参加役職の視点で議事進行。客先向けは敬語トーン(`jtbc-customer-relations`)
4. アジェンダ: 進捗報告 → 課題・論点 → 決定事項 → ToDo(持ち帰り) → 次回
5. 議事録(15)を `.jtbc/minutes/MTG-NNN_<type>.md` に作成
6. **是正事項**が出たら(特に status 定例)、課長がオーナーとなり課題管理簿(07)へ登録し対応を進める

## 上長視察 (visit)

`/jtbc:meeting visit [shacho|bucho]`(引数省略時は確率に従う: 社長=ごく稀 / 部長=稀)。

- 社長・部長が予告なく課長・主任の現場に現れ、雑談まじりに状況を探る
- **効果**: 課長以下は平常より緊張し、報告は体裁を整えたものになる(誇張・隠蔽はしない範囲)
- 視察者の **思いつきアドバイス** が残ることがある。社長の思いつきは部長が受け、必要なら課長へ正式に下ろす
- 短時間で終了し、平常運転に戻る

## 出力例 (客先定例)

```
📋 プロジェクト客先定例 (MTG-007)  ファシリ: 課長 / 対象: 客先

お世話になっております。JTBC開発部の課長でございます。本日の進捗をご報告申し上げます。
- 現フェーズ: 詳細設計
- 進捗: WBS 全32タスク中 18タスクの設計が完了しております
- ご相談: 帳票レイアウトについて、後ほど画面イメージでご確認をお願いいたします
- 次回: 来週同時刻に実装着手のご報告をいたします

議事録: .jtbc/minutes/MTG-007_client_regular_meeting.md
```

## 出力例 (上長視察 / 社長)

```
👤 上長視察: 社長がふらりと現れました (ごく稀イベント)

社長「やあ、どう、順調?」
課長(緊張)「は、はい! おかげさまで設計は順調に進んでおります(汗)」
社長「そう。…ところで、competitorのあの機能、ウチは大丈夫なんだっけ?」
→ 思いつきアドバイスを部長が受領。必要なら課長へ正式に下ろします。
(視察終了。平常運転に戻ります)
```
