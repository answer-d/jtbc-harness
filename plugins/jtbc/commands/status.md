# /jtbc:status

`.jtbc/state.json` を読んで現状を一覧表示します。
お客様向けにご報告する場合は `customer-relations` の敬語トーンに整えます。

## 実行手順

1. `.jtbc/state.json` を Read(無ければ `/jtbc:init` を案内)
2. 以下をまとめて表示:

```
JTBCプロジェクト状況

案件: <project_name> (<project_code>)   お客様: <client_name>
現フェーズ: <phase の日本語名>
体制: 部長(PJ責任者)/PMO/課長/主任/担当(<n>)   / 増員: 担当(部長承認で増減)

進行中の審査: <active_gate or "なし">
進行中の稟議: <active_ringi 一覧 or "なし">
進行中のWBSタスク: <active_wbs_task or "なし">
対応中のインシデント: <active_incidents 一覧 or "なし">

内部承認(ゲート)状況 ※客先提示の前提 / 起案者を除く承認者のみ表示・社長は非表示:
  - 提案審査:        [x] 部長  → 客先提示可
  - PJ計画審査:      [ ] 部長 / [ ] 主任
  - 基本設計審査:    [ ] 部長
  - 詳細設計審査:    [ ] 課長 / [ ] 部長
  ...

客先提示(ご査収)状況 ※内部承認の後に実施・ご承認で次フェーズへ:
  - 提案:        <APPROVED / PENDING / REVISION_REQUESTED / 未実施>
  - 要件定義書:  <...>
  - 基本設計書:  <...>
  - 詳細設計書:  <...>

次のアクション:
  → <推奨される次のコマンド>
```

3. フェーズに応じた推奨アクション(**内部承認 → 客先提示** の順):

| 現フェーズ | 推奨アクション |
|---|---|
| 提案 (PROPOSAL) | `/jtbc:hearing` で要望整理 → 課長が提案書起案 → (自動)社内審査 → (自動)客先提示・ご承認 |
| 要件定義 (REQUIREMENTS) | 課長が要件定義書・計画書 → (自動)社内審査 → (自動)客先提示・ご承認 |
| 基本設計 (BASIC_DESIGN) | 課長が基本設計 → (自動)社内審査 → (自動)客先提示・ご承認 |
| 詳細設計 (DETAILED_DESIGN) | 主任が詳細設計・WBS → (自動)社内審査 → (自動)客先提示・ご承認 |
| 実装 (IMPLEMENTATION) | 主任が担当へ割り振り、実装 → 完了確認で(自動)単体テストへ |
| 単体テスト (UNIT_TEST) | 担当が単体テスト → 完了確認で(自動)総合テストへ |
| 総合テスト (INTEGRATION_TEST) | 主任が総合テスト → (自動)リリース判定会 |
| リリース済 (RELEASED) | (自動)教訓整理 → (自動)PJ完了審査 |
| *_REVIEW | 審査会開催中(自動)。承認者の押印を待つ |

## インシデント発生中の表示

`active_incidents` が空でない場合は最上部に警告を出す:

```
⚠️ 緊急対応中: INC-001 「<title>」(severity: high)
　 司令塔がインシデント対応を進行中。状況は逐次ご報告いたします。
```

## エラーハンドリング

- `.jtbc/state.json` 不在 → `/jtbc:init` を案内
