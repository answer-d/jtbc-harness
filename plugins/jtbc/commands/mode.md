# /jtbc:mode

組織文化(Company Mode)の確認コマンド。

> 本プラグインは **JTBC 専用** です。かつて存在した startup モードは廃止しました。
> モードの切替(set)はありません。将来モードを追加する場合の拡張ポイントのみ残しています。

## サブコマンド

### `get`

現在の mode(常に jtbc)とその構成を表示。

```
現在のCompany Mode: jtbc (日本の伝統的大企業モード)

役職: 社長 / 部長 / 課長 / 主任 / 担当 / 外注SES
フェーズ: 提案 → 提案審査 → 要件定義 → PJ計画審査 → 基本設計 → 基本設計審査
        → 詳細設計 → 詳細設計審査 → 実装 → 単体テスト → 総合テスト
        → リリース判定会 → PJ完了審査 → 完了
ゲート: 6審査会 (提案/PJ計画/基本設計/詳細設計/リリース判定/PJ完了)
稟議: 5種類 (要件/設計/技術選定/スコープ/工数)
会議体: 定例4種 + 客先報告会議 + 上長視察
インシデント対応: あり (緊急報告→定期報告→なぜなぜ分析→障害報告書)
納品物: ソースコード + 各種設計書 + テスト結果 + リリース記録 + 教訓登録簿
```

### `list`

利用可能・将来候補の mode 一覧:

```
✅ jtbc   — 日本の伝統的大企業モード (本プラグインの唯一の実体)

将来の拡張候補(未実装・設計上の余地のみ):
   agile  — スクラム / PO+SM+Dev
   oss    — OSSコミュニティ / maintainer+contributor+reviewer
   gov    — 行政システム / 統括+設計+実装+監査
```

## 拡張アーキテクチャ (参考)

mode は `plugins/jtbc/modes/<mode>.yaml` で定義する設計。新文化を足すには yaml を追加し、
対応する役職 agent を `agents/` に置けばよい。ただし現状の実体は jtbc.yaml のみ。
