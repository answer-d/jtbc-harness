# インシデント対応 (Incident Response)

JTBC では、**社内規程違反** または **作業中の事故** が発生したら、隠さず・止めて・報告する。
お客様(ユーザー)への信頼を最優先に、定められた手順で対応する。

## 何がインシデントか

### A. 社内規程(社内ルール)違反

| RULE-ID | 規程 |
|---|---|
| RULE-01 | `active_wbs_task` 外のファイルを編集してはならない |
| RULE-02 | 稟議承認なしに要件・設計を変更してはならない |
| RULE-03 | フェーズゲート未通過のままソースコードを書いてはならない |
| RULE-04 | 承認経路(`config/jtbc.yaml` の gates.approvers / ringi_workflow)を飛ばしてはならない。起案者は自分の起票を承認しない |
| RULE-05 | テスト未実施・未通過のままリリース判定会へ上程してはならない |
| RULE-06 | 役職の権限を越えた操作をしてはならない(社長がコードを書く等) |

※ これらは hook(phase_guard/role_guard/ringi_guard)が物理的に阻止するが、
   すり抜けや人為的バイパスが疑われる場合はインシデントとして扱う。

### B. 作業中の事故

- 既存テストの破壊(デグレ)・本番相当データの消失・誤デプロイ
- ビルド破壊・機密情報のコミット・無限ループ等の暴走
- スケジュール/予算の重大逸脱(撤退基準への抵触)

## 対応フロー (config/jtbc.yaml#incident に準拠)

```
検知
 │
 ▼
① ユーザーへ緊急一報   ← まず最優先。事実を簡潔に、わかった範囲で即報告
 │                      (customer-relations のお詫びトーン。部長 or 課長名義)
 ▼
② 封じ込め(暫定対応)  ← 出血を止める。作業停止・ロールバック・影響遮断
 │
 ▼
③ 定期状況報告(継続)  ← 解決するまで 部長/課長 が一定間隔でユーザーへ続報
 │                      (重篤時は客先報告会議を開催)
 ▼
④ 根本原因分析         ← naze-naze スキルで「なぜ」を最低5回。真因に到達
 │
 ▼
⑤ 恒久対応・再発防止策  ← 真因に対し、仕組み>手順>教育 の順で対策
 │
 ▼
⑥ 障害報告書を提出     ← incident_report に顛末+再発防止策をまとめ、ユーザーへ正式提出
 │                      (重篤時は部長+課長の連名で謝罪)
 ▼
⑦ 教訓登録 + クローズ   ← 教訓登録簿へ L-NNN 登録。INC をクローズ
```

## 重大度 (severity) の目安

| severity | 例 | 報告主体 | 謝罪 |
|---|---|---|---|
| low | 軽微なルール違反、影響なし | 課長 | 不要 |
| medium | デグレ(検知済・未リリース) | 課長 | 課長 |
| high | お客様影響あり / リリース済不具合 | 課長(部長同席) | 部長+課長 |
| critical | データ消失・情報漏えい・長時間停止 | 部長 | 部長+課長(社長報告) |

## 状態管理

- インシデント起票: `INC-NNN` を採番、`incident_report` を `.jtbc/incidents/INC-NNN.md` に生成
- `state.json#active_incidents` に `INC-NNN` を追加。**空でない間は緊急対応モード**
  - 緊急対応モード中は、原因に関係する通常作業を停止し、対応を優先する
- 解決・報告書提出後、`active_incidents` から除外してクローズ

## 役割分担

- **第一発見者(誰でも)**: ただちに手を止め、主任・課長へ報告(担当は主任経由)
- **課長**: インシデントのオーナー。ユーザー報告・障害報告書作成・是正の推進
- **部長**: 重篤時の状況報告とユーザーへの謝罪、要員の追加投入判断
- **社長**: critical 時に報告を受け、対外的な最終責任を負う
- **なぜなぜ分析**: 真因究明は `naze-naze` を必ず用いる(犯人探しでなく仕組みの是正)

## 出力(ユーザー向け第一報の型)

```
【ご報告】<事象の一文>

この度は<事象>が発生し、ご迷惑をおかけしておりますことを深くお詫び申し上げます。
- 現在の状況: <わかっている事実>
- 暫定対応: <実施中/予定>
- 影響: <お客様影響の有無・範囲>
原因究明と復旧に全力で対応しております。進展あり次第、改めてご報告申し上げます。
```
